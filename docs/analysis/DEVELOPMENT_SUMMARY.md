# 开发总结

> 更新日期：`2026-04-28`

## 1. 当前版本的真实状态

项目当前已经不是“只完成数据抓取”的早期代码，也不是“多因子研究系统”的旧分支残留，而是一个可以直接启动的 Flask Web 应用。

现在的主线实现可以概括为：

- 服务端渲染页面
- 页面通过 `/api/*` 拉取结构化数据
- 数据主要来自 MySQL、本地缓存、Tushare、AkShare、LLM 服务
- 用户端、后台端、AI 会话、实时监控已经接入同一套应用工厂

## 2. 已收敛的功能面

### 2.1 用户端

- 市场概览首页
- 股票列表与筛选
- 股票详情分析
- 条件选股
- 策略回测
- 新闻聚合
- 实时监控
- AI 助手

### 2.2 用户体系

- 登录 / 注册 / 邮箱验证码
- 忘记密码
- 个人资料、密码、邮箱修改
- 自选股管理
- 分析记录管理
- 聊天记录查看
- 回测结果留存

### 2.3 管理端

- 管理员登录
- 用户分页管理
- 角色和状态调整
- 用户删除
- 系统日志查看
- 数据中心
- 单股分钟线同步
- 系统自检

## 3. 代码结构成熟度

### 3.1 已经比较稳定的部分

- `app/__init__.py` 的应用装配
- 页面、API、后台的 Blueprint 划分
- 股票详情、监控看板、新闻聚合、AI 会话这些主功能模块
- Session 鉴权 + 管理员权限守卫
- Redis 不可用时的缓存降级

### 3.2 仍需要统一口径的部分

- 回测逻辑分散在 `analysis_api.py` 与 `backtest_engine.py`
- AI 新旧记录表并存
- 财务三表工具在仓库里存在，但并不属于当前 Web 主链路
- 自动化测试缺失

## 4. 和旧文档相比，最重要的变化

本次整理前，部分旧文档仍然会误导维护者认为：

- ORM 模型总数还是 `22`
- 管理员用户列表没有分页
- 分析记录和回测结果还没有补齐
- 财务数据由 `scripts/sync_tushare_data.py` 统一驱动
- 旧 `ml_factor` 归档仍然有可读内容

这些说法现在都不准确了。

## 5. 目前最值得继续推进的工作

1. 为 `stock_api`、`analysis_api`、`auth_routes` 增加最小可用的 `pytest` 覆盖。
2. 清理 AI 会话与 `user_chat_history` 的双写设计。
3. 把当前激活的回测引擎提炼到独立 service，避免双实现。
4. 决定财务三表要么升级为正式能力，要么明确长期保留为脚本工具。
