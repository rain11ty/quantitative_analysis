# 数据库工具说明

> 更新日期：`2026-04-28`

## 1. 当前有哪些工具

仓库保留了两个手工数据库巡检脚本：

- `scripts/db_tools/database_explorer.py`
- `scripts/db_tools/db_viewer.py`

它们都不属于 `run.py` 的自动启动链路，而是供开发或排障时手工使用。

## 2. 适用场景

这些工具适合做下面几类事情：

- 看当前数据库里到底有哪些表
- 抽样检查某张行情表有没有数据
- 快速确认财务表、分钟线表、异动表是否同步成功
- 在改 ORM 或同步脚本前做人工核对

## 3. 使用方式

在项目根目录执行：

```bash
python scripts/db_tools/database_explorer.py
python scripts/db_tools/db_viewer.py
```

运行前请先确保 `.env` 中的数据库连接配置可用。

## 4. 不要把它们当成什么

这两个脚本不是：

- 自动化测试
- 正式运维平台
- 可直接暴露给终端用户的功能

它们的定位始终是“开发者手里的排障工具”。

## 5. 如果后续要升级

如果这个仓库后续要长期维护，建议下一步做的是：

1. 把稳定的巡检逻辑沉淀成 `pytest` 测试或统一 CLI 命令。
2. 把高频巡检项抽成只读 SQLAlchemy 查询脚本。
3. 把临时手工脚本与正式维护命令分层管理。
