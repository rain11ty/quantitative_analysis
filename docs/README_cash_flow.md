# 现金流量表脚本说明

> 更新日期：`2026-04-28`
> 对应文件：`app/utils/cash_flow.py`

## 1. 当前定位

`app/utils/cash_flow.py` 是一个独立脚本，用来创建并写入 `stock_cash_flow` 表。

和利润表、资产负债表不同的是，现金流量表当前在仓库里：

- 有脚本
- 有物理数据表逻辑
- 没有对应 ORM 模型
- 没有页面/API 主链路接入

所以它更像“历史保留的数据准备脚本”。

## 2. 它会做什么

脚本会：

- 创建 `stock_cash_flow`
- 从 Tushare 拉取现金流量表数据
- 批量插入 MySQL

脚本本身是一次性/批处理风格，不是服务化模块。

## 3. 如何运行

从项目根目录执行：

```bash
python app/utils/cash_flow.py
```

运行前需要确认：

- 数据库连通
- `.env` 已配置数据库参数
- `TUSHARE_TOKEN` 可用

## 4. 需要特别注意的地方

当前项目里：

- `stock_cash_flow` 不在 ORM 导出列表里
- `scripts/sync_tushare_data.py` 不会统一调度这个脚本
- 股票详情页、回测页、筛选页都不会直接读它

因此它不能被描述为“当前前台功能的一部分”。

## 5. 如果未来要正式接入

建议按下面顺序演进：

1. 先补 ORM 模型。
2. 把脚本迁移到 `scripts/` 并做参数化。
3. 设计 service 层查询。
4. 再决定接入哪些页面或 API。
